Part 1 - Build a Binary Classification Model

Section 1: Technical Details

The goal of this project is to classify the sentiment of text reviews (positive or negative) using structured features extracted from a dataset. This is achieved by preprocessing the data, training a regression-based classification model (ElasticNetCV), and evaluating its performance using the Area Under the Curve (AUC) metric. I used OpenAIs 1536-dimensional embeddings to train the classifier.

Data Preprocessing

The preprocess function handles the following steps:

  • Missing Value Imputation Missing values in the feature matrices are imputed using the most frequent value strategy via SimpleImputer. This ensures the data is complete for modeling.

Model Implementation: ElasticNetCV

  • Overview - ElasticNet is a regression technique that combines L1 regularization (Lasso) and L2 regularization (Ridge) to address multicollinearity and feature selection. It is particularly useful when the dataset has a mix of irrelevant and important features.

  • Hyperparameters -

    • alpha: Controls the overall regularization strength.
    • l1_ratio: Balances the proportion of L1 and L2 regularization.

  • Scaling the Data - ElasticNet requires standardized input features. The StandardScaler is used to scale the features to have zero mean and unit variance.

  • Training ElasticNet with Cross-Validation - The ElasticNetCV class performs hyperparameter tuning using cross-validation. The following settings are used:

    • l1_ratio: A list of values [0.1, 0.5, 0.9, 1.0] to explore different mixes of L1 and L2 regularization.
    • alphas: A range of values [0.001, 0.01, 0.1, 1.0, 10.0] to find the optimal regularization strength.
    • cv: 10-fold cross-validation to ensure robust evaluation.

The model selected the best combination of alpha and l1_ratio during training the model for each split. The best alpha and l1_ratio for each split is documented below.

  • Predictions and Probabilities ElasticNet produces continuous predictions, which are converted to probabilities using the sigmoid function (expit).

  • Model Evaluation The model is evaluated using the Area Under the Curve (AUC) metric for all the split from 1 to 5. Here are the AUC score for each split.

  • Model Saving I used pickle to save the ElasticNetCV model and scaler transformation done for Part 2 - Interpretability Analysis.

  • Execution Time Included execution time for each split in above table. On an average it took ~80s to process each split.

Part 2 - Interpretability Analysis

I did interpretability analysis on sentiment classification model trained on split1 dataset and used BERT-based sentence embeddings to interpret the results.

I tried to devide the process into a series of well-defined steps to ensure clarity and reproducibility. Below, each step is outlined with its purpose, implementation details, and relevance to interpretability.

Model binary and Embeddings of smaple positive and negative reviews are stored on GitHub at - https://github.com/harananikesh/PSL-CS598/tree/main

Detailed Explaination of Interpretability Approach

  • Sentence Segmentation

    • Purpose is to divide sample reviews into individual sentences, making it easier to attribute sentiment predictions to specific parts of the text

    • Implementation - The function segment_review utilizes the sent_tokenize method from the NLTK library to split a review into sentences. This enables sentence-level analysis rather than treating the entire review as a single unit.

  • Loading Pretrained Models

    • Purpose is to use a pretrained BERT model for generating high-dimensional contextual embeddings for the sample sentences.

    • Implementation - The load_bert function initializes the bert-base-uncased tokenizer and model from the Hugging Face Transformers library. These components are essential for encoding sentences into 768-dimensional vectors that capture their contextual meaning.

  • Generating Sentence Embeddings

    • Purpose is to convert each sentence into a fixed-length vector representation for downstream tasks such as sentiment prediction

    • Implementation - The get_sentence_embeddings function processes each sentence using the BERT tokenizer and extracts the embedding for the token, which serves as a representation of the entire sentence.

  • Dimensionality Alignment

    • Purpose is to ensure embeddings match the required dimensionality for compatibility with downstream models (e.g., 1536 dimensions).

    • Implementation - The align_embeddings function adjusts the dimensionality using padding or PCA. Padding adds zeros if the dimension is smaller, while PCA reduces the dimensionality if it is larger.

    • Saved the final embeddings for sample reviews [sentences] into GitHub for reproducibility.

  • Sentiment Prediction

    • Purpose is to predict the sentiment probability for each sample sentence embedding.

    • Implementation - The predict_sentence_sentiments function applies a scaler to normalize embeddings and uses a trained ElasticNet model to make predictions. The raw predictions are transformed into probabilities using the sigmoid function (expit).

  • Highlighting Key Sentences

    • Purpose is to visually interpret which sentences contribute most to the overall sentiment of the review.

    • Implementation - The highlight_sentences function assigns a background color to sentences based on their sentiment probabilities. Sentences with high probabilities (e.g., >0.5 for positive) are highlighted in green, while sentences with low probabilities are highlighted in red.

  • Load Artifects From Github

    • To reproduce the results, i have stored the trained model, scaler, and embeddings into GitHub repository.
    • Implementation - The load_from_github() load trained model, scaler and embeddings from Github.

  • Interpretability Analysis Workflow

    • Integrated the aforementioned steps into a coherent pipeline for interpretability analysis.
    • Implemented function - run_interpretability_analysis_v2()

def predict_sentence_sentiments(mymodel, scaler, embeddings):
  """Predict sentiment probabilities for each sentence embedding."""
  # Apply the scaler transformation to embeddings
  scaled_embeddings = scaler.transform(embeddings)
  #probabilities = mymodel.predict(scaled_embeddings)[:, 1]  # Assuming positive class is at index 1
  predictions = mymodel.predict(scaled_embeddings)  # 1D array of raw predictions
  probabilities = expit(predictions)  # Sigmoid to get values between 0 and 1
  return probabilities

#Highlight Key Sentences
def highlight_sentences(review, sentences, probabilities, threshold=0.5, prediction="positive"):
  """
  Highlight sentences contributing to the sentiment prediction.
  """
  highlighted_sentences = []
  for sentence, prob in zip(sentences, probabilities):
    highlighted = ""
    if (prediction == "positive" and prob > threshold) or (prediction == "negative" and prob < 1 - threshold):
      color = "rgba(0, 255, 0, 0.3)" if prediction == "positive" else "rgba(255, 0, 0, 0.3)"
      highlighted += f"<span style='background-color:{color}'>{sentence}</span> "
      highlighted_sentences.append(f">> {highlighted} (Probability: {prob:.3f})")
    else:
      highlighted += f"{sentence} "
      highlighted_sentences.append(f">> {highlighted} (Probability: {prob:.3f})")
  return highlighted_sentences

#Interpretability Analysis
def run_interpretability_analysis_v2():
  base_path = os.getcwd()

  random.seed(1088)

  print("load trained sentiment model from github")
  mymodel, myscaler, embeddings_results = load_from_github(base_path)
  embeddings_df = pd.DataFrame(embeddings_results)

  #Perform interpretability analysis for each review in the selected_reviews dataframe.
  results = []
  print("Running Interpretability Analysis for each positive and negative samples...")
  for i, row in embeddings_df.iterrows():
    review_id = row['review_id']
    review = row['review']
    sentiment = row['sentiment']
    sentiment_label = "Positive" if sentiment == 1 else "Negative"
    aligned_embeddings = row['embeddings']

    sentences = segment_review(review)
    probabilities = predict_sentence_sentiments(mymodel, myscaler, aligned_embeddings)
    interpretations = highlight_sentences(review, sentences, probabilities, threshold=0.5, prediction=sentiment_label.lower())
    results.append({
        "Review ID": review_id,
        "Sentiment": sentiment_label,
        "Highlighted Sentences": "\n".join(interpretations)
    })
  print("Completed Interpretability Analysis.")

  return results

Result Analysis

I have highlighted positive sentences with probability > 0.5 to Green and negative sentences with probability < 0.5 to Red.

With BERT-Base model embeddings, for negative reviews, model is performing pretty good. Probabilities are very low representiing high confidence in classifying reviews as negative.

Consider the Review ID: 19557 - Sentiment: Positive, model has low confidence on all of its sentences to be positive. Probably due to Complexity of Sentiments, for example - Sentences such as "I have actually received some spiritual strength from Northfork... Angels do exist..." and "Pay attention, as you'll need to use the brain and heart God gave you to follow the story." might not contain explicit markers of positivity (e.g., words like "amazing" or "great"). Instead, they convey subtle, subjective emotional and spiritual resonance.
